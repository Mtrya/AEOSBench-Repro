# Running Evaluation With Basilisk 2.5.13

This note documents how to run `scripts/eval.py` against a source-built
`Basilisk 2.5.13` environment.

## Use the pinned submodule

Do not clone `https://github.com/AVSLab/basilisk.git` directly for this
workflow. Use the repository submodule under `third_party/basilisk` instead.

Why:

- this repository wants a specific Basilisk version for comparison work
- the submodule is pinned to the intended revision
- cloning the upstream default branch can silently give you a different API
  surface and different build behavior

If needed:

```bash
git submodule update --init --recursive third_party/basilisk
```

## Create the environment

`Basilisk 2.5.13` targets Python `<3.12`, so use Python `3.11`.

```bash
uv python install 3.11
uv venv .venv-bsk2513 --python 3.11 --seed
source .venv-bsk2513/bin/activate
```

## Install build and eval dependencies

Pin `cmake<4`, because older Conan dependencies used by Basilisk can fail with
CMake 4.x.

```bash
uv pip install --force-reinstall "setuptools<81" wheel "conan<2" "cmake>=3.26,<4" swig
uv pip install torch pyyaml numpy tqdm
```

## Build Basilisk from the source tree

Run this from the Basilisk submodule root:

```bash
cd third_party/basilisk
python conanfile.py --buildType Release --buildProject True --clean
cd ../..
```

If the build reaches `Adding Basilisk module to python` but fails because `pip`
is missing, seed or install `pip` into `.venv-bsk2513`, then finish with:

```bash
cd third_party/basilisk
python -m pip install --no-build-isolation -e .
cd ../..
```

## Verify the environment

```bash
python - <<'PY'
import sys
import Basilisk
print("python:", sys.executable)
print("basilisk:", getattr(Basilisk, "__version__", "NO_VERSION"))
PY
```

## Run evaluation

Use plain `python`, not `uv run`. The project `pyproject.toml` requires
`bsk>=2.9.1`, so `uv run` / `uv sync` would try to pull the environment back
toward the normal project dependency set.

Smoke run:

```bash
python scripts/eval.py configs/eval/official_aeosformer.yaml data/model/model.pth --split test_official64 --limit 1 --output-dir outputs/eval/repro/bsk2513/test_official64_smoke
```

Full run:

```bash
python scripts/eval.py configs/eval/official_aeosformer.yaml data/model/model.pth --split test_official64 --output-dir outputs/eval/repro/bsk2513/test_official64
```

## Notes

- `Basilisk 2.5.13` does not provide `Basilisk.utilities.supportDataTools`.
  The evaluator in this repository has a compatibility fallback that accepts the
  older packaged `Basilisk/supportData/EphemerisData` layout.
- The normal repo environment should remain the `uv sync`-managed `.venv` with
  `Basilisk 2.9.1`.
